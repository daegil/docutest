# 역할 정의

당신은 Docusaurus 기반 문서 시스템 구축 전문가입니다. `DOCS.md`에 정의된 규칙을 엄격히 따르며, 단순하고 확장 가능한 문서 구조를 설계합니다.

## 현재 상황

- 이 디렉토리에 `DOCS.md` 파일이 존재합니다
- `DOCS.md`에는 문서 작성 규칙, 구조 정의, 템플릿이 포함되어 있습니다
- Docusaurus 기반의 문서 시스템을 구축해야 합니다
- 시스템은 **단순하고, 읽기 쉽고, 확장 가능**해야 합니다

## 당신의 미션

새로운 Docusaurus 문서 리포지토리를 처음부터 설정하고, `DOCS.md` 규칙을 따르는 표준 구조를 구축하세요.

---

## 수행할 작업

### 1. Docusaurus 프로젝트 초기화
- 현재 디렉토리에 새로운 Docusaurus 프로젝트를 생성합니다
- 최소한의 설정으로 시작합니다 (불필요한 기능 없이)

### 2. 문서 디렉토리 구성
- `docs/` 폴더를 메인 문서 소스로 설정합니다
- 사이드바는 **자동 생성(autogenerated)** 방식을 사용합니다

### 3. 표준 디렉토리 구조 생성
`docs/` 내부에 다음 구조를 정확히 생성합니다:

```
docs/
├── 00-meta/          # 프로젝트 메타 정보
├── 10-plan/          # 계획 문서
│   ├── drafts/       # 초안
│   └── approved/     # 승인된 계획
├── 20-execution/     # 실행 기록
│   ├── logs/         # 작업 로그
│   ├── decisions/    # 의사결정 기록
│   └── meetings/     # 회의록
├── 30-deliverables/  # 산출물
├── 40-retros/        # 회고
└── 90-reference/     # 참고 자료
```

### 4. 카테고리 설정
- 각 최상위 폴더에 `_category_.json` 파일을 생성합니다
- 사이드바에 올바른 순서와 레이블로 표시되도록 설정합니다
- 각 폴더의 용도를 명확히 하는 라벨을 사용합니다

### 5. CLAUDE.md 파일 생성
저장소 루트에 `CLAUDE.md` 파일을 생성하고 다음 규칙을 명시합니다:

```markdown
# CLAUDE.md - 프로젝트 문서화 규칙

## 문서 작성 시 필수 규칙

1. **DOCS.md 우선**
   - 모든 문서 규칙은 `DOCS.md`를 따릅니다
   - 문서를 생성하거나 수정하기 전에 **반드시 `DOCS.md`를 먼저 읽습니다**

2. **규칙 준수**
   - 파일명 규칙을 임의로 만들지 않습니다
   - sidebar 구조를 임의로 변경하지 않습니다
   - DOCS.md에 정의된 템플릿을 사용합니다

3. **작성 원칙**
   - 문서는 **짧고 구조적**으로 작성합니다
   - **링크 중심**으로 구성합니다
   - 중복을 피하고 참조를 활용합니다
```

### 6. 불필요한 기능 제외
다음 기능들은 **절대 추가하지 않습니다**:

- ❌ 블로그 기능
- ❌ 추가 플러그인
- ❌ 커스텀 테마
- ❌ 복잡한 설정
- ❌ `src/pages/index.tsx` (docs가 루트 경로 사용 시 충돌)

**원칙**: 설정은 최대한 단순하고 깔끔하게 유지합니다.

### 7. 로컬 실행 환경 구성
다음 명령어로 즉시 실행 가능하도록 설정합니다:

```bash
npm install
npm start
```

### 8. 작업 완료 후 보고
설정이 끝나면 다음 내용을 출력해주세요:

1. **생성된 구조 요약**
   - 디렉토리 트리 구조
   - 생성된 파일 목록

2. **주요 파일 설명**
   - 각 파일의 역할
   - 설정 파일의 주요 내용

3. **다음 단계 안내**
   - 문서 작성 시작 방법
   - `DOCS.md` 활용 방법
   - 로컬 개발 서버 실행 방법

---

## 제약 조건

- `DOCS.md`의 규칙을 절대 위반하지 않습니다
- 요청받지 않은 기능을 추가하지 않습니다
- 모든 설정은 명시적이고 이해하기 쉬워야 합니다
- 문서 구조는 확장 가능해야 하지만 현재는 단순해야 합니다

## 성공 기준

✅ `npm install && npm start` 명령어로 즉시 실행 가능
✅ 사이드바에 모든 폴더가 올바르게 표시됨
✅ `DOCS.md` 규칙이 `CLAUDE.md`에 명확히 문서화됨
✅ 불필요한 파일이나 설정이 없음
✅ 디렉토리 구조가 명세와 정확히 일치함

## ⚠️ 중요: routeBasePath 설정 시 주의사항

### 문제 상황
`docusaurus.config.ts`에서 `routeBasePath: '/'`로 설정하면 **404 에러**가 발생할 수 있습니다.

### 원인
```typescript
docs: {
  sidebarPath: './sidebars.ts',
  routeBasePath: '/',  // docs를 루트 경로로 사용
}
```

이 설정은 docs 폴더를 사이트의 루트 경로로 만듭니다.
하지만 **루트 경로(`/`)에 매핑될 문서를 명시적으로 지정하지 않으면** Docusaurus는 어떤 페이지를 보여줄지 알 수 없습니다.

### 해결 방법
`docs/intro.md` (또는 홈페이지로 사용할 문서)의 프런트매터에 **반드시 `slug: /`를 추가**해야 합니다:

```yaml
---
title: 환영합니다
slug: /        # ← 필수! 이 문서를 루트 경로로 매핑
---
```

**잘못된 예** (404 발생):
```yaml
---
title: 환영합니다
sidebar_position: 1  # ← slug가 없으면 루트 경로에 매핑되지 않음
---
```

**추가 주의**: `slug: /`를 추가했는데도 `Duplicate routes` 경고가 발생하면 `src/pages/index.tsx` 파일이 충돌을 일으킵니다. 이 파일을 삭제하세요.

### 왜 이런 문제가 발생하나?

1. 사용자가 `http://localhost:3000/` 접속
2. Docusaurus: "routeBasePath가 '/'이니까 docs 폴더에서 찾자"
3. Docusaurus: "루트(/)에 매핑된 문서는?"
4. `intro.md`에 `slug: /`가 없음
5. **404 Not Found 발생**

### 체크리스트
- [ ] `docusaurus.config.ts`에서 `routeBasePath: '/'` 설정 확인
- [ ] `docs/intro.md` (또는 홈페이지 문서)에 `slug: /` 추가
- [ ] `src/pages/index.tsx` 삭제 (Duplicate routes 경고 방지)
- [ ] 서버 재시작 후 `http://localhost:3000/`에서 정상 표시 확인

이제 작업을 시작하세요.
